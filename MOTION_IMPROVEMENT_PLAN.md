# 个人网站视觉与交互动效优化计划

## 目标

在不改变现有信息架构、内容展示和跳转下载功能的前提下，对当前个人网站进行小范围视觉与用户交互优化。优化重点是让页面在保持现有工业风、网格感、红黑配色和卡片式布局的基础上，增加更克制、更一致、更有高级感的动效表现。

本计划不追求全站大面积动画，也不将页面改造成视频化展示。动效只服务于重点模块的信息表达和交互反馈。

## 基本原则

- 保持现有风格：延续当前的米白背景、黑红强调色、硬朗阴影、网格背景、chamfered badge、Archivo / Space Grotesk / DotGothic16 字体气质。
- 小范围增强：重点优化工具栈模块，少量优化 FAQ、下载按钮和联系入口，不对 Hero、作品经历等核心内容做大范围重构。
- 不影响展示功能：所有现有文本、链接、PDF 下载、外链跳转、FAQ 展开逻辑都必须保持可用。
- 动效克制：避免全站同时出现大量 pulse、ping、漂浮、旋转等抢注意力的动画。
- 性能优先：优先使用 transform、opacity、box-shadow、filter 等适合浏览器渲染的属性，避免频繁触发布局重排。
- 可访问性优先：考虑 `prefers-reduced-motion`，为减少动态效果的用户保留稳定体验。

## Remotion Skill 的应用方式

当前项目是 React 网站，已有 `framer-motion`，不是 Remotion 视频工程。因此不建议直接引入 Remotion 库来驱动网页交互。更合理的方式是借鉴 Remotion 的动效设计方法，并用现有 React + Framer Motion 实现。

可借鉴的 Remotion 思路：

- 时间轴思维：把动效拆成「容器出现、边框响应、图标反馈、文字状态变化」几个阶段，而不是所有元素同时变化。
- 插值思维：将鼠标位置、滚动进度、打开状态映射到旋转、位移、透明度、光泽位置等视觉属性。
- 统一缓动：使用类似 `cubic-bezier(0.16, 1, 0.3, 1)` 的缓出节奏，形成一致的页面动效语言。
- 分层编排：重要模块使用 staggered reveal，普通模块保持简单 hover，避免动效权重失衡。

## 优化范围

### 1. 工具栈模块：重点优化

位置：`THE ARSENAL` 工具栈卡片区。

目标：将工具栈模块做成全站最有交互记忆点的区域，但不破坏当前 bento card 结构。

建议实现：

- 卡片根据鼠标位置产生轻微 3D tilt，幅度控制在可读性范围内。
- 卡片内部增加柔和光泽层，随鼠标位置移动，模拟高质量 hover glare。
- 图标容器增加轻微 magnetic 效果，鼠标靠近时产生小幅偏移和缩放。
- 工具图标 hover 不再只是 `scale-110`，增加边框点亮、背景微变、图标轻微旋转。
- 工具项之间仍保持当前分隔线和列表结构，不改内容层级。
- 父级卡片进入视口时使用 staggered reveal，让四张工具卡片按顺序出现。

验收标准：

- 工具栈模块在桌面端鼠标移动时有明确但克制的互动反馈。
- 鼠标离开后卡片能平滑回到初始状态。
- 移动端不依赖 hover，不出现错位或卡片倾斜残留。
- 工具名称、描述、图标全部保持可读。

### 2. FAQ 模块：轻量优化

位置：`PM PHILOSOPHY` FAQ 展开区。

目标：保留当前手风琴交互，只优化展开和收起的质感。

建议实现：

- 保留当前点击展开逻辑和默认打开第一项的行为。
- 展开时答案内容增加轻微 `opacity + y` 过渡。
- 箭头旋转使用统一缓动，避免机械感。
- 当前激活项的红色边框和阴影保持，但减少突兀跳变。
- 问题标题可增加轻微 underline reveal 或 active 状态强调。

验收标准：

- 点击每个 FAQ 项仍能正确展开和收起。
- 同一时间仍只打开当前选中的 FAQ。
- 展开动画不导致内容闪烁、跳动或布局异常。

### 3. 下载按钮与联系入口：微反馈优化

位置：页面底部 `CONTACT` 区域。

目标：让下载简历、下载作品集、邮箱和电话入口更像可交互控件。

建议实现：

- 下载按钮 hover 时增加背景 wipe 或阴影位移，但保留当前强对比按钮风格。
- 下载图标 hover 时做轻微下沉，强化“下载”语义。
- 邮箱和电话 hover 增加 underline reveal 或图标轻微偏移。
- 不新增复杂提交表单，除非后续明确需要联系表单功能。

验收标准：

- 两个 PDF 下载链接仍能正常下载。
- 邮箱 `mailto:` 和电话 `tel:` 仍保持原功能。
- 动效不影响按钮文字阅读，不造成误触或布局位移。

### 4. 全局动效规范：小幅整理

目标：减少动效配置分散问题，为后续维护提供统一入口。

建议实现：

- 在 `App.tsx` 中抽出统一 motion presets，例如 `sectionReveal`、`cardReveal`、`iconHover`、`faqTransition`。
- 将重复的 `whileInView`、`transition`、`viewport` 配置归一。
- 减少 CSS `transition-all` 的使用，改为更明确的属性过渡。
- 保留现有 Tailwind 工具类，但避免 CSS 动画和 Framer Motion 对同一元素重复控制 transform。

验收标准：

- 页面视觉效果与当前版本保持同一风格方向。
- 代码中的动效配置更集中、更易维护。
- 不引入不必要的新依赖。

## 非目标

- 不重做整个网站设计。
- 不替换现有技术栈。
- 不新增完整联系表单，除非后续单独确认。
- 不引入 Remotion 作为网页运行时动效库。
- 不制作全屏开场动画、滚动驱动长动画或视频背景。
- 不修改简历、作品集 PDF 文件内容。

## 实施顺序

1. 整理全局 motion presets，建立统一缓动和动效节奏。
2. 优先实现工具栈模块的鼠标互动效果。
3. 调整 FAQ 展开动画和 active 状态反馈。
4. 优化下载按钮和联系入口 hover 反馈。
5. 做响应式和 reduced-motion 检查。
6. 运行构建和 lint，确认没有破坏现有功能。

## 验收清单

- `npm run build` 可以通过。
- `npm run lint` 无新增错误，若存在历史问题需明确标注。
- 桌面端工具栈模块有鼠标互动效果，且不卡顿。
- 移动端布局和内容展示不受 hover 动效影响。
- 所有导航锚点仍能跳转到对应区块。
- 所有外链仍使用新窗口打开。
- 邮箱、电话、PDF 下载链接功能保持不变。
- FAQ 展开收起功能保持不变。
- 页面没有大面积、持续性、干扰阅读的动画。
- 视觉风格仍与当前红黑工业风个人站一致。

## 风险与控制

- 风险：过度使用 3D tilt 可能显得花哨。
  控制：旋转幅度控制在小范围内，并只应用于工具栈模块。

- 风险：hover 动效和 CSS `transition-all` 冲突。
  控制：后续实现时将关键交互统一交给 Framer Motion 管理。

- 风险：移动端没有 hover，互动效果失效。
  控制：移动端保留静态视觉和点击功能，不强行模拟鼠标动效。

- 风险：动效影响性能。
  控制：优先使用 transform 和 opacity，避免频繁更新布局属性。

## 后续可选方向

如果后续希望真正使用 Remotion，可以将其作为独立视频资产生产工具，而不是网页交互库。例如：

- 为个人网站生成 5-10 秒开场宣传视频。
- 为作品集项目生成产品演示短片。
- 将工具栈能力图谱导出为社媒传播用 MP4/GIF。

这些内容可作为独立工程或独立脚本处理，不建议混入当前网站运行时。
